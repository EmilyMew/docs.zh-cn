# <a name="ihosttaskmanagerbegindelayabort-method"></a>IHostTaskManager::BeginDelayAbort 方法
通知宿主托管代码进入不能在其中中止当前任务的周期。  
  
## <a name="syntax"></a>语法  
  
```  
HRESULT BeginDelayAbort ();  
```  
  
## <a name="return-value"></a>返回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`BeginDelayAbort` 已成功返回。|  
|HOST_E_CLRNOTAVAILABLE|公共语言运行时 (CLR) 尚未加载到进程中，或处于不能运行托管的代码或已成功处理调用的状态。|  
|HOST_E_TIMEOUT|呼叫已超时。|  
|HOST_E_NOT_OWNER|调用方不拥有该锁。|  
|HOST_E_ABANDONED|事件已取消时被阻塞的线程或纤程正在等待它。|  
|E_FAIL|发生未知的灾难性故障。 如果某方法返回 E_FAIL，CLR 不再在进程内可用。 对托管方法的后续调用返回 HOST_E_CLRNOTAVAILABLE。|  
|E_UNEXPECTED|`BeginDelayAbort` 已调用，但相应地调用[EndDelayAbort](../../../../docs/framework/unmanaged-api/hosting/ihosttaskmanager-enddelayabort-method.md)尚未收到。|  
  
## <a name="remarks"></a>备注  
 主机必须中止当前任务之前`EndDelayAbort`调用。 如果另一个调用`BeginDelayAbort`对的干预调用不建立`EndDelayAbort`，主机应返回 E_UNEXPECTED 从`BeginDelayAbort`，并且应采取任何操作。
