# <a name="iclrtaskneedspriorityscheduling-method"></a>ICLRTask::NeedsPriorityScheduling 方法
获取一个值，该值指示当前任务，正被切换出，是否需要将其标记为高优先级以便重新计划。  
  
## <a name="syntax"></a>语法  
  
```  
HRESULT NeedsPriorityScheduling (  
    [out] BOOL *pbNeedsPriorityScheduling  
);  
```  
  
## <a name="parameters"></a>参数  
 `pbNeedsPriorityRescheduling`  
 [out]`true`，如果主机应尝试尽可能快; 否则为重新计划当前的任务实例`false`。  
  
## <a name="return-value"></a>返回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`NeedsPriorityRescheduling` 已成功返回。|  
|HOST_E_CLRNOTAVAILABLE|公共语言运行时 (CLR) 尚未加载到进程中，或处于不能运行托管的代码或已成功处理调用的状态。|  
|HOST_E_TIMEOUT|呼叫已超时。|  
|HOST_E_NOT_OWNER|调用方不拥有该锁。|  
|HOST_E_ABANDONED|事件已取消时被阻塞的线程或纤程正在等待它。|  
|E_FAIL|发生未知的灾难性故障。 如果某方法返回 E_FAIL，CLR 不再在进程内可用。 对托管方法的后续调用返回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>备注  
 在其中的任务是接近于收集的垃圾回收器的情况下，CLR 设置的值`pbNeedsPriorityScheduling`到`true`，指示重新安排高优先级。 这允许主机快速重新计划任务，从而最大程度减少垃圾回收中出现延迟的可能性，并使宿主和运行时能够共同协作来节约内存资源。
